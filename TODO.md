## 2. 프롬프트 설계 — 5개 카테고리별 규칙 정리

- [x] **2-1 회사명 치환** — 최종 설계: "AI는 분류만, 찾기·치환은 코드가 결정적으로 처리"
  - **1차 설계(폐기)**: AI에게 "본문에서 회사명이 나오는 위치를 찾고, 문맥별로 치환 여부까지 판단해라"라고 맡겼더니, 같은 회사명이 여러 문장에 반복될 때 일부를 놓치는 버그 발생 (실제 테스트: 4곳 중 2곳만 잡힘). "찾기"와 "판단"을 동시에 시키는 게 원인 — 찾기는 AI가 완벽히 보장할 수 없는 작업.
  - **최종 설계**: 역할을 분리한다.
    1. AI는 본문에 언급된 회사명을 나열하고, 각각을 `worked_at`(경력 설명 문맥) / `target`(지원동기 등 그 외 문맥 — 치환 대상)으로만 분류한다. "몇 번 나오는지"는 신경 쓰지 않아도 됨 — 회사명당 한 번만 판단.
    2. 코드(`buildCompanySuggestions()`)가 `target`으로 분류된 회사명에 대해, 본문을 문장 단위로 쪼갠 뒤 그 문자열이 포함된 **모든 문장을 결정적으로(문자열 검색) 찾아** 각각 치환 제안을 만든다. 개수 제한 없음, 절대 누락되지 않음.
  - **전제(팀 논의로 확정)**: 지원하는 회사와 이미 근무한 회사의 이름은 절대 같을 수 없다. 본문에 예시·비교 목적으로 언급된 회사명은 없다고 가정한다 (즉 `worked_at`이 아니면 전부 `target`). 이 전제 덕분에 `target` 회사명은 문맥 재판단 없이 발견되는 즉시 전부 치환해도 안전함.
  - **실제 API 테스트로 검증**: 회사명이 5개 문장에 걸쳐 반복되고 경력 문맥 1곳이 섞인 샘플에서, 경력 문맥은 그대로 유지되고 나머지 5곳 전부 개별 제안(`company`, `company-1` ~ `company-4`)으로 정확히 생성됨을 확인
  - **남은 리스크**: AI가 본문의 회사명 자체를 회사명으로 인식하지 못하는 경우(드문 케이스)는 여전히 놓칠 수 있음 — 실용적으로 충분하다고 판단해 현재는 추가 조치 없음
- [ ] **2-2 인재상·가치 제안**
  - 본문에서 기존에 지원했던 회사의 인재상/가치 관련 문장을 탐지하는 기준 정의
  - 목표 회사의 mission/value는 AI 자체 지식으로 추론 (0번 결정사항) — "모르면 일반론으로 답하고 단정하지 마라" 제약 문구 포함, `confidence` 필드로 추정 여부 표시
- [ ] **2-3 직군·포지션 치환**
  - 직군/포지션 키워드 탐지 기준 정의 (직무명, 관련 업무 키워드 등)
  - `role` 파라미터로 정확히 치환
- [ ] **2-4 문법·맞춤법 제안**
  - "선택 가능하도록" 이라는 요구사항 반영 — 대안이 여러 개일 수 있는지, 아니면 1건 제안 후 사용자가 적용/미적용만 선택하는 기존 UX(`sugg__toggle`)로 충분한지 확인
- [ ] **2-5 문장 톤 제안**
  - 목표 회사 서비스 톤도 AI 자체 지식으로 추론 — 2-2와 동일한 할루시네이션 제약/`confidence` 표시 적용
- [ ] 모든 카테고리 공통: 자소서 본문에 실제로 존재하지 않는 내용을 지어내지 않도록(할루시네이션 방지) 명시적 제약 문구 추가
- [ ] 프롬프트에 "반드시 JSON만 응답" 등 출력 형식 강제 문구 포함 (마크다운 코드펜스 없이)

## 9. 알려진 이슈 (추후 처리)

- [ ] **제안 겹침(overlap) 문제**: 같은 문장이 여러 카테고리(company/keyword/position/grammar/tone)에서 동시에 제안될 수 있음. `applySuggestions()`가 제안을 순서대로 하나씩 `.replace(original, suggestion)` 하는 구조라서, 앞 순서 카테고리(예: company)가 먼저 그 문장을 바꿔버리면, 뒤 순서 카테고리(예: grammar)의 `original`이 더는 본문에 존재하지 않아 **사용자가 적용을 눌러도 조용히 무시**되는 상황이 생길 수 있음.
  - 실제 테스트에서 확인: "카카오의 백엔드 개발자로서 안정적인 서비스를 만들고 싶습니다." 문장이 `company`, `position`, `grammar` 세 카테고리에서 동시에 잡힘
  - 하이라이팅(문제 1) 작업과 함께 다룰 예정 — 겹치는 구간을 어떻게 표시/적용할지 정책 결정 필요 (하나만 우선할지, 순서 재조정할지, 적용 시 경고를 줄지 등)
